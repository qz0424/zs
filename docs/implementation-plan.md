# 实施计划 - 窗帘定制下单系统

## 当前状态
- 产品展示 + 收藏/对比 + 管理端款式管理 ✅
- 无订单系统

## Phase 1：订单核心（客户下单 + 管理端接单）

### 1.1 数据库新增表
- `orders` — 订单主表（客户信息+地址+状态）
- `order_items` — 订单项（每组窗帘：款式+颜色+尺寸+备注）
- `order_status_log` — 状态变更日志
- `settings` — 商家设置（手机号/微信号/二维码/Webhook）
- 验证：orders + order_items 一起保存在事务中

### 1.2 后端 API 新增
| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/orders` | 客户提交订单（无需登录） |
| GET | `/api/orders` | 管理端订单列表（支持筛选） |
| GET | `/api/orders/:id` | 订单详情（含items+日志） |
| PUT | `/api/orders/:id/status` | 管理端切换订单状态 |
| GET | `/api/settings` | 获取商家设置（公开） |
| PUT | `/api/settings` | 更新商家设置（需登录） |

### 1.3 客户端下单页
- 新增"下单"导航入口（底部导航栏或款式详情页按钮）
- 地址表单：姓名、手机号（必填）、小区（下拉+搜索）、楼栋、单元、房号
- 窗帘组管理：空间下拉（固定选项）、款式选择（图片+名称）、颜色选择、宽高输入（数字+单位）、备注
- 【添加另一处窗户】按钮 → 新增一组，带序号标记
- 每组可单独删除
- 提交前预览汇总页

### 1.4 管理端订单管理
- 新增"订单"标签页
- 订单列表：订单号、客户、地址、状态、时间、操作
- 筛选：按状态、小区、时间范围
- 订单详情弹窗：左侧地址信息，右侧分条展示每组窗帘
- 一键切换状态按钮

### 1.5 实时通知
- 客户提交订单 → 管理端通过 Server-Sent Events (SSE) 推送
- 管理端显示弹窗 + 未读小红点
- 企业微信/钉钉 Webhook 可选配置

**验证标准：**
- 客户可完成完整下单流程
- 管理端可查看订单列表和详情
- 状态切换正常工作
- 通知实时到达

## 技术决策
- **订单号生成**：`OD` + 日期 + 序列号，如 `OD20260628001`
- **实时通知**：SSE（Server-Sent Events），比 WebSocket 简单，适合单向推送
- **地址管理**：小区名称由管理端预设（在 settings 表中存储常用小区列表），客户从下拉框选择
- **数据一致性**：订单和订单项在同一事务中保存
- **权限**：订单提交无需登录（匿名下单），订单管理需登录
