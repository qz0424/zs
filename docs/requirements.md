# 窗帘定制下单系统 - 需求文档

## 1. 产品定位
- **产品名称：** 窗帘定制下单系统
- **目标用户：** 客户（门店顾客、业主）、管理员（工厂/门店老板）
- **使用场景：** 客户浏览款式 → 选择搭配 → 提交定制订单 → 管理端接单生产
- **设备平台：** 手机（安卓 + iOS）+ 电脑网页
- **扩展性：** 管理端联系方式可自定义，支持推广给其他商家使用

## 2. 核心功能

### 2.1 客户端（客户用）
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 款式浏览 | 主图+多色归组，支持视频，搜索/筛选 | P0 |
| 收藏对比 | 收藏感兴趣款式，多款对比参数 | P0 |
| **下单** | **填写地址+手机号，选款式/颜色/尺寸，提交定制订单** | **P0** |
| 多组窗帘 | 一个订单可添加多处窗户，每组独立选款式/颜色/尺寸/备注 | P0 |
| 订单预览 | 提交前汇总清单，客户核对 | P0 |
| 历史订单 | 查看历史下单记录，一键复用地址 | P1 |
| 联系商家 | 首页展示商家手机号、微信号、二维码 | P0 |

### 2.2 管理端（管理用）
| 功能 | 说明 | 优先级 |
|------|------|--------|
| 款式管理 | 新增、编辑、下架款式（已有） | P0 |
| **订单管理** | **查看所有订单，按状态/小区/时间筛选** | **P0** |
| **订单详情** | **左侧客户地址，右侧分条展示每组窗帘** | **P0** |
| **状态流转** | **新下单→已核对→生产中→已完工→已配送，一键切换** | **P0** |
| **实时通知** | **客户提交订单后管理端弹窗+小红点提醒** | **P0** |
| **联系方式设置** | **管理端可配置手机号、微信号、微信二维码** | **P0** |
| 打印订单 | 订单清单可打印，给车间生产用 | P1 |
| 分类管理 | 自定义分类维度和选项（已有） | P0 |
| 批量导入 | Excel批量导入款式数据（已有） | P0 |

## 3. 数据结构

### 3.1 订单（orders）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | number | 订单编号 |
| order_no | string | 订单号（如 OD20260628001） |
| customer_name | string | 客户姓名 |
| customer_phone | string | 手机号（必填） |
| community | string | 小区/楼盘名称（必填） |
| building | string | 楼栋号 |
| unit | string | 单元 |
| room | string | 房间号 |
| address_detail | string | 补充地址（如门牌号等） |
| status | string | new/confirmed/producing/completed/delivered |
| note | string | 整单备注 |
| created_at | datetime | 下单时间 |
| updated_at | datetime | 更新时间 |

### 3.2 订单项（order_items）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | number | 记录ID |
| order_id | number | FK → orders |
| sort_order | number | 序号（1号客厅、2号主卧...） |
| space | string | 空间（客厅/主卧/次卧/儿童房/阳台） |
| curtain_id | number | FK → curtains（选择的款式） |
| color_id | number | FK → curtain_colors（选择的颜色） |
| width | number | 宽度（米） |
| height | number | 高度（米） |
| quantity | number | 数量（默认1） |
| note | string | 单组备注（加厚、遮光、特殊挂钩等） |

### 3.3 订单状态日志（order_status_log）
| 字段 | 类型 | 说明 |
|------|------|------|
| id | number | 记录ID |
| order_id | number | FK → orders |
| from_status | string | 原状态 |
| to_status | string | 新状态 |
| changed_at | datetime | 变更时间 |

### 3.4 商家设置（settings）
| 字段 | 类型 | 说明 |
|------|------|------|
| key | string | 设置键（phone/wechat/wechat_qr/webhook_url等） |
| value | string | 设置值 |
| updated_at | datetime | 更新时间 |

## 4. 客户下单流程

```
浏览款式 → 选中款式 → 点击"下单"
    ↓
填写联系信息（姓名、手机号、小区地址）
    ↓
添加窗帘组（空间 + 款式 + 颜色 + 尺寸 + 备注）
    ↓
点击【添加另一处窗户】→ 新增一组
    ↓
订单预览（汇总清单：地址 + 各房间窗帘详情）
    ↓
确认提交 → 后台弹窗通知
```

## 5. 订单状态流转

```
新下单 (new) → 已核对 (confirmed) → 生产中 (producing) → 已完工 (completed) → 已配送 (delivered)
```

每个状态变更记录日志，客户可在客户端查看进度。

## 6. 商家联系方式

管理端设置页面可配置：
- 手机号（客户端首页展示）
- 微信号（客户端首页展示）
- 微信二维码图片（客户端首页展示）
- 企业微信/钉钉 Webhook URL（订单通知）

## 7. 避坑规则
- 多组窗帘绑定同一张订单，不每组单独生成订单
- 所有选择（空间、款式）用固定选项，不自由输入
- 客户端和管理端数据实时同步
- 地址结构化（小区+楼栋+单元+房号），不用自由文本
